# References

## OpenFeature

* [Introduction | OpenFeature](https://openfeature.dev/docs/reference/intro) — Official documentation for an open specification that provides a vendor-agnostic, community-driven API for feature flagging, covering core concepts such as SDKs, providers, and hooks.

## Feature Flag Review Criteria

* [Feature Flag Review Criteria](./feature-flag-review-criteria.md) — An index for phase-specific Feature Flag review criteria.
* [Feature Flag Design Review Criteria](./feature-flag-design-review-criteria.md) — Criteria for reviewing Feature Flag designs, ADRs, plans, requirements, governance, lifecycle, security boundaries, and over-engineering risk.
* [Feature Flag Implementation Review Criteria](./feature-flag-implementation-review-criteria.md) — Criteria for reviewing Feature Flag implementation code, evaluator behavior, security checks, observability, tests, and complexity.

## Literature on Feature Flags

* [Feature Toggles (aka Feature Flags)](https://martinfowler.com/articles/feature-toggles.html) — A comprehensive guide by Pete Hodgson (published on martinfowler.com) that categorizes toggles into four types (Release, Experiment, Ops, Permission) and covers implementation patterns and operational trade-offs.
* [LaunchDarkly Engineering Blog](https://launchdarkly.com/blog/) — Engineering articles from a feature flag platform provider covering practical topics such as flag lifecycle management, gradual rollouts, and reducing flag debt.
